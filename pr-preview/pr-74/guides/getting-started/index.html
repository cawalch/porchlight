Getting Started

Porchlight is a single CSS file with zero JavaScript dependencies. It ships as a complete design system: tokens, themes, base styles, layout primitives, 34 components, and progressive enhancements.

Install

npm install @cawalch/porchlight
# or
pnpm add @cawalch/porchlight

Import

The package main export is the prebuilt stylesheet — a single file with no @import statements to resolve:

@import "@cawalch/porchlight";

That’s it. The framework self-registers all cascade layers, tokens, and components. No configuration, no build step, no JavaScript.

Bundler users: see Bundler notes below if you use esbuild or Bun, which cannot parse some modern CSS features in the framework. Vite works out of the box.

The layer model

Porchlight wraps everything in a single top-level cascade layer: @layer porchlight. Inside it, nine sub-layers declare a deterministic order:

@layer porchlight {
  @layer reset, tokens, themes, base, layout,
         components, utilities, enhancements, print;
}

Later layers win over earlier ones, so a component always beats base, a utility always beats a component, and nothing needs !important.

Host overrides

To override Porchlight from your application CSS, place your own layer after the framework:

/* app.css */
@layer porchlight, app; /* app wins */
@import "@cawalch/porchlight";

@layer app {
  /* your product styles here */
  .my-button {
    /* overrides .c-button */
  }
}

Bundler caveat: some bundlers (esbuild, Bun) hoist @import above the @layer declaration, reversing the intended order. If this happens, import Porchlight from your JS entry instead:

// app.ts
import "@cawalch/porchlight";
import "./app.css";

Then put only the @layer declaration in app.css (no @import to hoist). See Bundler notes below.

Unlayered styles always beat layered styles too, so plain CSS outside any @layer also wins. You never fight !important: Porchlight reserves it for accessibility-critical utilities only ([hidden], screen-reader-only helpers, reduced-motion).

Do not pass layer(...) to the @import. Each Porchlight module self-wraps in @layer porchlight.<name>; adding layer(...) would double-nest it.

Using components

Each component is a class prefix: .c-*. Drop them into your HTML:

<button class="c-button" data-variant="primary">Save</button>

<div class="c-card">
  <div class="c-card__header">
    <h2 class="c-card__title">Title</h2>
  </div>
  <div class="c-card__body">Content</div>
</div>

Browse the component reference or the preview gallery to see every component with live examples.

Using layout primitives

Layout helpers use the .l-* prefix:

<div class="l-grid" style="--l-grid-min: 16rem;">
  <div class="c-card">A</div>
  <div class="c-card">B</div>
  <div class="c-card">C</div>
</div>

Using utilities

Utilities use the .u-* prefix. A small, finite set:

<span class="u-muted">Secondary text</span>
<span class="u-truncate">very-long-filename.webp</span>
<span class="u-sr-only">Screen reader only</span>
<a class="u-focusable-sr-only" href="#main">Skip to content</a>

Browser support

Porchlight targets Baseline 2025+ browsers. The core (layers, custom properties, :has(), @scope, color-mix(), OKLCH) works in all modern browsers. Advanced features (anchor positioning, scroll-driven animations, interpolate-size) are @supports-gated and degrade gracefully.

See the Browser Support guide for the full feature matrix.

Bundler notes

Porchlight uses modern CSS features (@property, @scope, @container scroll-state()) that are Baseline 2025+ but not yet understood by all CSS parsers. Here’s how to handle common bundlers:

Vite

Works out of the box — Vite uses Lightning CSS which handles all features natively.

esbuild / Bun

esbuild and Bun’s built-in CSS parsers throw a hard error on @property, @scope, and @container scroll-state(). Two workarounds:

Option A — copy the prebuilt file to your static assets:

cp node_modules/@cawalch/porchlight/dist/porchlight.css ./static/porchlight.css
<link rel="stylesheet" href="/porchlight.css" />

Option B — import from JS and exclude from the bundler’s CSS pipeline:

// Exclude porchlight from CSS processing
import "@cawalch/porchlight/dist/porchlight.css?inline";

Either way, declare your layer order separately in your app CSS:

@layer porchlight, app;
