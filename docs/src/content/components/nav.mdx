# Nav

The `.pl-c-nav` renders a vertical navigation list for sidebars. Icon + label
items with active state via `aria-current="page"`. Collapsible sub-sections
via native `<details>`. Icons-only variant for collapsed sidebars.

It also includes compact sidebar footer patterns for SaaS apps: metadata such
as account identity, workspace, tenant, environment, or session status, plus
small full-width actions that can be rendered as either links or buttons.

For nav-launched menus, compose `.pl-c-menu` with `.pl-c-nav__menu`. Use this for
actions, flyouts, and destination lists. Use `.pl-c-dropdown` only when the nav
control is choosing a value rather than opening a navigation/action menu.

## Semantic HTML

```html
<nav class="pl-c-nav" aria-label="Main">
  <a class="pl-c-nav__item" aria-current="page">
    <svg class="pl-c-nav__icon">...</svg>
    <span class="pl-c-nav__label">Dashboard</span>
  </a>
  <div class="pl-c-nav__group">
    <span class="pl-c-nav__group-label">Workspace</span>
  </div>
  <details class="pl-c-nav__section" open>
    <summary class="pl-c-nav__section-trigger">
      <svg class="pl-c-nav__icon">...</svg>
      <span>Projects</span>
      <svg class="pl-c-nav__chevron">...</svg>
    </summary>
    <a class="pl-c-nav__item pl-c-nav__item--child">
      <span class="pl-c-nav__label">Active</span>
    </a>
  </details>
</nav>
```

## Groups and Dividers

Use `.pl-c-nav__group-label` for quiet, non-interactive grouping. Use
`.pl-c-nav__divider` when a separator is enough and a label would add noise.

```html
<nav class="pl-c-nav" aria-label="Main">
  <div class="pl-c-nav__group">
    <span class="pl-c-nav__group-label">Workspace</span>
  </div>
  <a class="pl-c-nav__item" href="/dashboard">Dashboard</a>
  <a class="pl-c-nav__item" href="/accounts">Accounts</a>
  <hr class="pl-c-nav__divider" />
  <a class="pl-c-nav__item" href="/settings">Settings</a>
</nav>
```

## Nav Menus

Use `.pl-c-nav__menu` on a `.pl-c-menu` wrapper when a sidebar row opens a menu or
flyout. The trigger uses `.pl-c-menu__trigger`, but Porchlight gives that trigger
the same density, icon alignment, current state, and icon-rail behavior as
other nav rows.

```html
<nav class="pl-c-nav" aria-label="Main">
  <div
    class="pl-c-menu pl-c-nav__menu"
    data-placement="inline-end"
    style="--pl-c-menu-anchor: --nav-reports;"
  >
    <button
      class="pl-c-menu__trigger"
      type="button"
      popovertarget="nav-reports"
      aria-haspopup="menu"
    >
      <svg class="pl-c-nav__icon" aria-hidden="true">...</svg>
      <span class="pl-c-nav__label">Reports</span>
      <svg class="pl-c-nav__chevron" aria-hidden="true">...</svg>
    </button>
    <div class="pl-c-menu__popover" id="nav-reports" popover>
      <a class="pl-c-menu__item" href="/reports" aria-current="page">
        <span class="pl-c-menu__item-body">
          <span class="pl-c-menu__item-label">Overview</span>
          <span class="pl-c-menu__item-description">Executive KPI rollup</span>
        </span>
        <kbd class="pl-c-menu__item-shortcut">R</kbd>
      </a>
      <button class="pl-c-menu__item" type="button">Export CSV</button>
    </div>
  </div>
</nav>
```

`data-placement="inline-end"` places the menu beside the sidebar row, which is
the usual desktop rail/flyout pattern. Keep `aria-haspopup="menu"` on triggers
that open action menus. If your app adds roving focus, typeahead, or submenu
keyboard behavior, own that JavaScript in the app layer; the CSS kit keeps
native links and buttons reachable by Tab.

## Footer metadata and actions

Use `.pl-c-nav__footer` at the end of a sidebar nav for compact metadata. The
metadata block keeps long labels truncated, and `.pl-c-nav__action` provides a
full-width action surface for links or buttons without the weight of a regular
`.pl-c-button`.

```html
<nav class="pl-c-nav" aria-label="Main">
  ...
  <div class="pl-c-nav__footer">
    <div class="pl-c-nav__meta">
      <span class="pl-c-nav__meta-icon">
        <svg aria-hidden="true">...</svg>
      </span>
      <span class="pl-c-nav__meta-body">
        <span class="pl-c-nav__meta-label">avery.long.email@example.com</span>
        <span class="pl-c-nav__meta-description">
          <span class="pl-c-badge" data-tone="success">Admin</span>
          Online
        </span>
      </span>
    </div>
    <div class="pl-c-nav__actions">
      <a class="pl-c-nav__action" href="/switch-user">
        <svg class="pl-c-nav__action-icon" aria-hidden="true">...</svg>
        <span class="pl-c-nav__action-label">Switch user</span>
      </a>
      <button class="pl-c-nav__action" type="button">
        <svg class="pl-c-nav__action-icon" aria-hidden="true">...</svg>
        <span class="pl-c-nav__action-label">Sign out</span>
      </button>
    </div>
  </div>
</nav>
```

The footer metadata selectors are intentionally generic. Use them for account
or session state when that is your product language, but the public API does
not require the metadata to represent a user account.

## Icons-only (collapsed sidebar)

```html
<nav class="pl-c-nav" data-variant="icons" aria-label="Quick nav">
  <a class="pl-c-nav__item" aria-current="page" aria-label="Dashboard">
    <svg class="pl-c-nav__icon">...</svg>
  </a>
  <div class="pl-c-menu pl-c-nav__menu" data-placement="inline-end">
    <button
      class="pl-c-menu__trigger"
      aria-label="Reports"
      popovertarget="reports"
    >
      <svg class="pl-c-nav__icon">...</svg>
      <span class="pl-c-nav__label">Reports</span>
    </button>
    <div class="pl-c-menu__popover" id="reports" popover>...</div>
  </div>
</nav>
```

In icon-only nav, every trigger or link needs a durable accessible name
(`aria-label`, visible tooltip trigger text, or equivalent). Labels, badges,
group labels, and chevrons are visually hidden by the variant.

## Class contract

| Selector                      | Role                             |
| ----------------------------- | -------------------------------- |
| `.pl-c-nav`                   | Container (flex column).         |
| `.pl-c-nav__item`             | A navigation link.               |
| `.pl-c-nav__item--child`      | Nested item inside a section.    |
| `.pl-c-nav__icon`             | Leading icon.                    |
| `.pl-c-nav__label`            | Text label.                      |
| `.pl-c-nav__badge`            | Trailing badge (count).          |
| `.pl-c-nav__group`            | Non-interactive item group.      |
| `.pl-c-nav__group-label`      | Quiet group label.               |
| `.pl-c-nav__divider`          | Separator between item groups.   |
| `.pl-c-nav__menu`             | `.pl-c-menu` wrapper inside nav. |
| `.pl-c-nav__footer`           | Bottom metadata/action region.   |
| `.pl-c-nav__meta`             | Compact metadata block.          |
| `.pl-c-nav__meta-icon`        | Metadata icon/avatar slot.       |
| `.pl-c-nav__meta-body`        | Metadata text stack.             |
| `.pl-c-nav__meta-label`       | Truncated metadata label.        |
| `.pl-c-nav__meta-description` | Secondary metadata row.          |
| `.pl-c-nav__actions`          | Compact action stack.            |
| `.pl-c-nav__action`           | Small full-width link/button.    |
| `.pl-c-nav__action-icon`      | Action icon slot.                |
| `.pl-c-nav__action-label`     | Truncated action label.          |
| `.pl-c-nav__section`          | Collapsible `<details>` group.   |
| `.pl-c-nav__section-trigger`  | The `<summary>` (click target).  |
| `.pl-c-nav__chevron`          | Expand/collapse arrow.           |
| `[aria-current]`              | Current item, except `"false"`.  |
| `[data-variant="icons"]`      | Icons-only mode (hides labels).  |
